# Ontology Design Simulator

一个用于本体（Ontology）建模与语义查询联动的可视化模拟器：在中心画布中构建对象类型与关系网络，通过“语义查询端口”输入自然语言或结构化语句，系统会流式给出解析结果，并在底部以多种协议/表示形式展示可执行的语义预览（如 RDF/OWL/SWRL/Query DSL/GraphQL），用于沟通、校验与后续集成。

## 适用场景
- 领域建模：把业务对象、属性与关系以图谱方式结构化表达
- 语义联动：从自然语言到结构化操作意图、实体识别与字段生成
- 多协议预览：同一语义意图在不同语义/查询表示中的映射与对照
- 模拟验证：基于 GraphQL 模板与变量，触发后端模拟接口获取回执与结果

## 界面核心模块（5 个）

### 1) 顶部状态与操作栏（Header / Top Bar）
- 全局入口与状态：项目/画布状态提示、快捷按钮等

### 2) 左侧资产导航区（Left Asset Sidebar）
- 资产/流程导航：语义层/动态层/静态层切换、操作/实体列表、数据流与流程列表

### 3) 中心图谱画布（Central Graph Canvas）
- 建模主工作区：对象类型节点、关系连线与画布交互（拖拽、缩放、布局）

### 4) 右侧语义查询面板（Right Semantic Query Panel）
- 自然语言输入与解析：输入语句、触发解析、查看解析状态与结构化结果
- 本体配置解析区：解析提示、识别实体与置信度、生成字段（用于操作所需参数）等

### 5) 底部代码与协议预览区（Bottom Code Preview Panel）
- 本体语句解析区：以多 Tab 形式预览语义场景、RDF/Turtle、OWL、本体规则（SWRL）、Query DSL、GraphQL 等
- 用于复核与对齐：在协议/语句层面确认“系统理解是否正确”

## 协作与迭代
- 需求/验收看板：使用仓库根目录的 [plan.md](./plan.md) 记录需求卡片（用户故事 + 验收标准），并按 Backlog / In Progress / Done 维护状态流转
